## What v0.4.0 Delivered

### §1 Token-Tier Invariant (spec §18)

Token usage decomposes into three top-level tiers `C7_eager_per_session /
C8_oncall_per_trigger / C9_lazy_avg_per_load` (beside `metrics` and
`core_goal`; NOT inside R6 D2). Adds OPTIONAL EAGER-weighted
`iteration_delta` formula `weighted_token_delta = 10×eager + 1×oncall +
0.1×lazy`, `R3` split into `R3_eager_only` / `R3_post_trigger`, Informative
`prose_class` taxonomy, `lazy_manifest` packaging gate, and `tier_transitions`
block on `iteration_delta_report.yaml`. Si-Chip itself is degenerate at the
EAGER axis (C7 = 0; pure on-demand loading) — Cycle 2 chip-usage-helper R30
yielded -55% EAGER win, which is the canonical evidence base. spec_validator
BLOCKER 12 `TOKEN_TIER_DECLARED_WHEN_REPORTED` enforces top-level block when
any tier sub-axis is reported.

### §2 Real-Data Verification (spec §19)

Normative sub-step of §8.1 step 2 `evaluate` (the main 8-step list count
unchanged). Three layers:

* **msw fixture provenance** (the `// real-data sample provenance:
  <captured_at> / <observer>` grep-able comment pattern).
* **user-install verification** (run the ability against real production
  payloads at install time; not just unit-test fixtures).
* **post-recovery live verification** (after any rollback, re-run against
  real-data samples to confirm recovery).

`templates/feedback_real_data_samples.template.yaml` formalizes the
production-sample-pull schema (REDACTED before commit). spec_validator
BLOCKER 13 `REAL_DATA_FIXTURE_PROVENANCE` enforces fixture-citation when
ability declares real_data_samples.

### §3 Stage Transitions & Promotion History (spec §20)

§2.2 stage enum gains a formal `stage_transition_table` (reverse transitions
forbidden; e.g., `productized → exploratory` is not allowed). Adds
`BasicAbility.lifecycle.promotion_history` as an append-only audit trail
of state transitions, and `metrics_report.yaml.promotion_state` as a
first-class top-level block. When `round_kind == 'ship_prep'`, the round
emits a **7th evidence file** `ship_decision.yaml` (vs base 6); spec_validator
EVIDENCE_FILES BLOCKER is round_kind-aware.

### §4 Health Smoke Check (spec §21)

`BasicAbility.packaging.health_smoke_check` declares pre-ship probes against
4 axes `{read, write, auth, dependency}`. OPTIONAL at schema level, REQUIRED
when `current_surface.dependencies.live_backend: true` (hard rule 13 /
BLOCKER 14). §8.1 step 8 `package-register` runs every declared probe and
gates ship-eligibility. Probes emit OTel spans `gen_ai.tool.name=si-chip.health_smoke`
(spec-internal observability; does NOT reinvent Kubernetes
liveness/readiness). Si-Chip itself sets `live_backend: false` (no live
runtime dependency at install time); chip-usage-helper Cycle 5 R47 was the
canonical evidence base.

### §5 Eval-Pack Curation Discipline (spec §22)

Minimum pack size by gate: v1 20 prompts, v2 **40 prompts** (curated
near-miss bucket REQUIRED), v3 60+ recommended. G1 `_provenance ∈
{real_llm_sweep, deterministic_simulation, mixed}` is first-class REQUIRED
on `metrics_report.yaml` (was OPTIONAL in v0.3.0). Informative
`templates/eval_pack_qa_checklist.md` codifies the curator's checklist.
Normative `templates/recovery_harness.template.yaml` formalizes T4
error_recovery_rate measurement. §3.2 frozen constraint #5: deterministic
seed `hash(round_id + ability_id)`. Real-LLM cache directory at
`.local/dogfood/<DATE>/<round_id>/raw/real_llm_runner_cache/`.

### §6 Method-Tagged Metrics (spec §23)

Every R6 sub-metric emits a `<metric>_method` companion field. Token metrics
take `{tiktoken, char_heuristic, llm_actual}`; quality/routing metrics take
`{real_llm, deterministic_simulator, mixed}`; G1 takes `{real_llm_sweep,
deterministic_simulation, mixed}`. `_method == char_heuristic` also requires
`_ci_low` + `_ci_high` 95% confidence-interval bands. New observability
add-ons: `U1_language_breakdown` (en/zh/mixed_warning) and
`U4_time_to_first_success_state ∈ {warm, cold, semicold}`.
`spec_validator::R6_KEYS` BLOCKER ignores companion suffixes. The
chip-usage-helper R36 2.4× error evidence (Flesch-Kincaid against zh prose
miscounts) was the canonical evidence base.

### §7 The Real-LLM Runner (FIRST production unblock)

`evals/si-chip/runners/real_llm_runner.py` is the **first production real-LLM
runner in the Si-Chip codebase** — 884 LoC + 27 tests. Components:

* `RouterFloorAdapter` — maps spec router_floor labels (`composer_2/fast`,
  `sonnet_shallow`) to physical endpoint/model pairs (`claude-haiku-4-5`,
  `claude-sonnet-4-6`) per r12.5 §5.1 MODEL_REMAP rule (composer_2 is
  Cursor-internal; not reachable from sandbox shell).
* `AnthropicMessagesClient` — raw `requests.post` against Veil-egressed
  `/v1/messages`; preserves spec §11 forever-out boundary (NO weight updates,
  NO online learning, NO router-model training).
* `RealLlmRunner.evaluate_pack` / `evaluate_router_matrix` — runs k=4
  sampling per prompt against the mvp 8-cell matrix (2 models × 2 depths × 2
  packs).
* Cache directory per §22.6 — content-addressed by `hash(model + depth +
  prompt + sample_idx + seed)`; `--seal-cache` flag for CI determinism per
  r12.5 §5.2.

The deterministic SHA-256 simulator's `pass_k_4 = pass_rate^4 = 0.7222^4 =
0.5478` PROXY (a lower bound) had blocked v2_tightened T2 row since Round 1
by -0.0022. Round 18 ran 640 calls × 130 tokens ≈ 86k input tokens + 2.5k

of 8 cells: all near_miss cells); mean across cells = 0.9719. Round 19
cache replay at 100% hit ($0; ~20ms wall-clock). The PROXY blocker is now
formally retired.
