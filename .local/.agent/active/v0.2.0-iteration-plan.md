## Risks and mitigations (consolidated)

- **Round 3** — router_floor cells are simulated (deterministic); R6 number is a sweep latency, not a real-LLM latency. Documented Round 4+ upgrade path. Acceptable at v1_baseline.
- **Round 4** — `L4_redundant_call_ratio` may be degenerate (0.0) in the 6 simulated cases. Surface honestly; do not fudge.
- **Round 5** — `U1` Flesch-Kincaid grade may be unflattering on the bilingual SKILL.md `description`. Surfacing the number is the value.
- **Round 6** — `U4` install timing depends on network latency to the tarball; record p50 + p95 separately if variance is high.
- **Round 7** — `C5` context-rot risk is a heuristic; record as `v1_baseline`-acceptable proxy, not ground truth.
- **Round 8** — `V2` credential_surface scan must NOT log secret values; only counts and pattern names. Workspace "No Silent Failures" applies if secrets are detected.
- **Round 9** — Adding an `intermediate` profile changes a §9 Factory Template Contract file (`router_test_matrix.template.yaml`). Bump its `$schema_version` to 0.1.1 and ensure validator accepts both 0.1.0 and 0.1.1 (backward-compat).
- **Round 10** — `G1` from a 2-model × 2-pack matrix is a partial generalizability proxy; informative not authoritative. Full G1 needs the 96-cell matrix at v2_tightened+.
- **Round 11 (spec bump)** — must update `.rules/.compile-hashes.json` AND `.local/research/` pointer AND `AGENTS.md` AND every template's `$schema_version` reference. Drift detection (`.rules/check-rules-drift`) must pass after the bump or the round is invalid. Strongly prefer **backward-compat schema** (validator accepts both 0.1.0 and 0.2.0-rc1 evidence files) so prior rounds' evidence remains valid.
- **Round 12 (ship verification)** — if `consecutive_v2_passes < 2` at Round 12 close, v0.2.0 ship is BLOCKED. The orchestrator must spawn a Round 13/14 to satisfy spec §4.2 promotion rule before tagging v0.2.0.
